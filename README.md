## Project Overview

Developed for an "Inclusive UX/UI" course, focusing on accessibility for visually impaired users. 

Learning a Universal Gaming Experience that integrates accessibility features for the visually impaired directly into game, ensuring seamless interaction for all users regardless of physical ability. 

---
## How to Play
The gameplay revolves around listening to a sequence of animal sounds and accurately rearranging them to progress.

Goal : Get highest level player can achieved.

- Step 1 : Player need to listern or visually observe to a sequence of animal sounds

- Step 2 : Player tap animal to rearrange sound accurately or swipe to select and tap to animal incase of using Screen Reader (Visual impaired accessibility) 
- Step 3 : After accurately rearrangeing, Player can go to next level

## Built With
Unity 6000.3.4f1, Unity Built-in Accessibility API

This project utilizes components from the [Unity Accessibility Public Sample](https://github.com/Unity-Technologies/a11y-public-sample)  by Unity Technologies.
## Key Learning

- **Inclusive Engineering:** Mastered Universal Design to create a unified experience for all users, rather than developing segregated platforms for the visually impaired.
- **Accessibility Tech:** Gained proficiency in Unity’s Built-in Accessibility tools, ensuring seamless compatibility with TalkBack and VoiceOver screen readers. 
- **User-Centric Iteration:** Translated qualitative research from user interviews into technical features like Haptic Feedback and Non-Visual Cues.
- **Advanced Navigation:** Developed custom Gesture-based Input (Swipes) that operates without conflicting with standard OS Screen Reader commands.
- **Software Architecture:** Applied the Singleton Pattern to build a modular, scalable Manager system for efficient global access to core game logic.
- **Empathy-Driven QA:** Adopted Empathy Testing protocols, such as blindfolded playtesting, to validate the entire game loop through audio and touch alone.
